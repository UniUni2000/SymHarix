# Code of Conduct

SymHarix is meant to be useful, serious engineering software. The project works
best when people can ask questions, disagree, and improve the system without
personal attacks.

## Expected Behavior

- Be respectful and direct.
- Assume good intent while still being precise about risks and bugs.
- Give actionable feedback with enough context to reproduce or understand it.
- Keep discussions focused on the project and the work.
- Protect private information, credentials, logs, and user data.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or exclusionary behavior.
- Publishing private information, credentials, or security-sensitive details.
- Repeated disruption of issues, pull requests, or discussions.
- Misrepresenting test results, security impact, or project behavior.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests;
or block contributors when behavior harms the project or its contributors.

Security-sensitive reports should follow `SECURITY.md`, not public discussion
threads.
